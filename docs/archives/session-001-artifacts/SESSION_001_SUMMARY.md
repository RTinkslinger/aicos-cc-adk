## THE CORE INSIGHT: Network as Operating System

### The Problem
Aakash's competitive advantage is his network — 1,000–2,000 active relationships. He is an outbound-first networker who actively researches and finds people. This is rare among VCs.

**The system breaks down after the meeting:**
1. **Follow-ups fall through cracks** — #1 pain point
2. **Too many meeting requests, hard to prioritize** — #2
3. **Relationship status untracked** (who haven't I spoken to?) — #3
4. **Context prep before meetings takes too long** — #4

**New interactions mostly stay in memory — unlogged.** No structured system exists between "I met this person" and "I should reconnect because X."

### The Reframe
The AI CoS is NOT primarily a deal flow tool or portfolio tracker. It is a **Network Operating System**. Everything flows from the network:
- Deal flow = function of network health
- Portfolio support = relationship management with a different label
- IC collective = specific subnet of the network
- Content = network signaling (attracts and activates relationships)

### Relationship Categories Identified (Preliminary — NOT FINAL)
1. Angel portfolio founders (100+ from personal investing)
2. DeVC portfolio founders (140–150/cycle)
3. Z47 portfolio founders (~45/cycle)
4. Prospective founders (active deal pipeline)
5. Beyond-strike-zone founders (deal flow + learning sources)
6. Future founders (operators today, potential founders tomorrow)
7. Founder-angels (potential IC additions)
8. Independent Capitalists (50+ DeVC collective)
9. Other VCs (co-invest, competitive intel)
10. LPs (current + prospective)
11. Operators/executives (hiring network for portcos)
