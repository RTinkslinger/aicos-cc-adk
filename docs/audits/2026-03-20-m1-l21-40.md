**Changes:**
- `types.ts`: Added `ActionScoreRankedRow` and `ActionScoreBreakdownRow` interfaces
- `queries.ts`: Added `fetchActionScoresRanked()` and `fetchActionScoreBreakdown(actionId)` queries
- `ScoreBreakdownBars.tsx`: New component — horizontal bars for 6 factors + strategic composite, percentile badges, days-old indicator
- `ActionDetailSheet.tsx`: Fetches breakdown on-demand via server action, shows ScoreBreakdownBars (compact mode)
- `actions/[id]/page.tsx`: Fetches breakdown in parallel, shows ScoreBreakdownBars with fallback to old scoring
- `ActionsBucketList.tsx`: Shows blended_score (color-coded) in card footer, passed via lookup from page
- `ActionsPageClient.tsx`: Passes blendedScoreLookup through to bucket list
- `actions/page.tsx`: Fetches `fetchActionScoresRanked()` in parallel, builds lookup map

**DB views used:**
- `action_scores_ranked` — id, action, blended_score, type_percentile, global_percentile
- `action_score_breakdown` — id, f_time, f_stakeholder, f_conviction, f_effort, f_novelty, f_irgi, strategic_score, days_proposed, staleness_boost

**Score display:** blended_score >= 7 = green, >= 4 = amber, else muted

## L29-32: Supabase Realtime for /actions

**What:** Added live notification when new actions are inserted while user is on the actions page.

**Changes:**
- `client.ts`: New browser-side Supabase client using `NEXT_PUBLIC_SUPABASE_PUBLISHABLE_DEFAULT_KEY`
- `useRealtimeActions.ts`: Hook subscribing to `postgres_changes` INSERT on `actions_queue`
- `ActionsPageClient.tsx`: Shows cyan "N new actions — tap to refresh" bar that calls `router.refresh()` on click

**Graceful degradation:** If publishable key is missing or realtime fails, the hook logs a warning and returns 0 — no UI shown.

**Dependency:** Requires `NEXT_PUBLIC_SUPABASE_PUBLISHABLE_DEFAULT_KEY` env var to be set on Vercel.
