## L21-24: Queue Processor Routing Fix (CRITICAL)

### Root Cause
`process_cir_queue()` dequeued messages and logged them to `cir_propagation_log`, but **never called `process_cir_queue_item()`**. All 3,672 messages processed before this fix were logged-only with zero propagation.

Additionally, `process_cir_queue_item()` expected `target_action` values (`rescore_actions`, `refresh_company`, `refresh_network`) that didn't match what the triggers actually send (`rescore_related_actions`, `refresh_company_connections`, `refresh_person_connections`, `process_new_action`, `process_new_interaction`, `update_thesis_indicators`, `update_preference_weights`).

### Fix Applied
1. **`process_cir_queue()`** rewritten to call `process_cir_queue_item(msg.message)` for every dequeued message
2. **`process_cir_queue_item()`** rewritten to handle all 7 `target_action` values from `cir_propagation_rules`:
   - `rescore_related_actions` / `rescore_actions` -> `rescore_related_actions(thesis_id)`
   - `update_thesis_indicators` -> `update_thesis_indicators(thesis_id)`
   - `process_new_action` -> `compute_user_priority_score(action_row)` (fixed signature: takes row, not int)
   - `update_preference_weights` -> logged (M5 handles actual weight updates)
   - `process_new_interaction` -> upserts `cir_state` for interaction tracking
   - `refresh_person_connections` / `refresh_network` -> `propagate_network_change(person_id)`
   - `refresh_company_connections` / `refresh_company` -> `propagate_company_change(company_id)`

### Sub-fix: compute_user_priority_score signature
The `process_new_action` handler originally called `compute_user_priority_score(integer)`, which doesn't exist. The actual function signature is `compute_user_priority_score(action_row actions_queue)`. Fixed to SELECT the row first, then pass it.

### Test Results
- Synthetic company update (id=1): routed to `propagate_company_change(1)` -- PASS
- Synthetic thesis update (id=1): routed to `update_thesis_indicators(1)`, returned indicators JSONB -- PASS
- Synthetic network update (id=1): routed to `propagate_network_change(1)` -- PASS
- All results logged with `processing_result` in `actions_taken` JSONB

## L37-40: Performance Optimization

### Batch Deduplication
Created `process_cir_queue_batch()` -- a smarter queue processor that:
- Tracks seen entities per batch via array
- Skips duplicate events for same entity within a batch (common during bulk updates)
- Logs batch timing in milliseconds
- Logs `BATCH_DEDUP` events for skipped duplicates
- Logs `BATCH_SUMMARY` with processed/errors/deduped/batch_ms

### Cron Updated
- Replaced `process_cir_queue()` with `process_cir_queue_batch()` in cron schedule
- Job runs every minute (unchanged schedule)

### New Functions
- `prune_weak_connections(threshold)` -- deletes connections below threshold, logs to propagation log
- `connect_orphaned_entities(batch_size)` -- creates embedding-similarity connections for unconnected entities
