### Cascade Fix (Critical)
**Root cause:** `cir_actions_queue` trigger fired on ALL INSERT/UPDATE on actions_queue, with no WHEN condition. When `propagate_network_change` updated `strategic_score`, it fired the CIR trigger, which generated more queue events, creating a feedback loop.
**Fix:** Dropped and recreated `cir_actions_queue` as INSERT-only (removed UPDATE). Actions queue data changes are now only tracked on initial insert, not on score recalculations. Other triggers (network, companies, interactions) were already column-specific and safe.
