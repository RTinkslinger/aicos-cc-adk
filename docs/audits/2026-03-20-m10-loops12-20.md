### FINDING: Queue Processor Does Not Route to Handler Functions

**Severity: MEDIUM**

`process_cir_queue()` (cron job 5, runs every minute) reads messages from the queue, logs them to `cir_propagation_log`, and archives them. It does **NOT** call `process_cir_queue_item()` which contains the actual routing logic to:
- `rescore_related_actions()` for thesis changes
- `propagate_company_change()` for company changes
- `propagate_network_change()` for network changes

The propagation functions exist and work (tested), but the queue processor does not invoke them. Messages are logged but not acted upon.

**Impact:** Changes are detected and logged but downstream propagation (rescoring, connection refresh) only happens when functions are called directly — not via the automated queue pipeline.

**Fix:** Replace the body of `process_cir_queue()` to call `process_cir_queue_item(msg.message)` for each dequeued message.
