## Connection Pool Lessons

Mass UPDATE operations trigger embedding + CIR triggers, which spawn pgmq messages. With ~5,000 records updated, the 10-second embedding cron consumed all available connections for 60-90 seconds per batch. Mitigation:
1. Batch UPDATEs to 500-1,000 rows max
2. Wait 30-45 seconds between batches for pool recovery
3. Embedding queue self-heals; no intervention needed beyond patience
