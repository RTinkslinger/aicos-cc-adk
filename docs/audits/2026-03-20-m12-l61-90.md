### L69-72: Email Enrichment
- Created **derive_email_candidates()** function: generates 6 email patterns per person from name + company domain (first.last, first, firstlast, flast, firstl, last.first).
- Created **email_candidates** table (11 columns) for storing derived candidates pending verification.
- Generated **4,024 email candidates** for **673 people** across **356 domains**.
- Set first.last@domain pattern as derived email on 673 network records.
- **Result: Email coverage 0% -> 19.1%** (673/3,525 network people).
- All derived emails marked with `enrichment_status = 'derived_email'` and `enrichment_source = 'datum_email_derivation'`.
- Note: These are unverified derived emails. Verification requires MX check or SMTP verification (future loop).
