### Verified Behaviors
1. **EMA learning (lr=0.3)** correctly shifts preference weights:
   - Meeting/Outreach: 0.050 -> 0.133 (+0.083, positive shift from 4,5,4,3 ratings)
   - Portfolio Check-in: -0.200 -> -0.020 (+0.180, strong positive swing from rating=5)
   - Research: 0.175 -> 0.063 (-0.112, negative pull from rating=2)
   - Pipeline/Deals: new at 0.400 (first rating=5)
   - Thesis Update: new at -0.400 (first rating=1)

2. **action_outcomes table** correctly populated with 10 new rows (IDs 10-19), outcome labels mapped from 1-5 scale

3. **Subsequent scores shift** after preference updates: re-normalization correctly incorporates the new preference weights

### Feedback Loop Architecture
```
User Rating (1-5) -> update_preference_from_rating(action_id, rating)
  -> Maps rating to adjustment: (rating - 3.0) * 0.2
  -> EMA update: new_adj = (1-lr)*old_adj + lr*adjustment (lr=0.3)
  -> Writes to preference_weight_adjustments (4 dimensions: action_type, priority, source, thesis)
  -> Records outcome in action_outcomes table
  -> Next normalize_all_scores() run picks up new preference weights
```

**Overall hierarchy: Ultra > Investigate > Scan approximately equals Skip -- CORRECT**

### Problem Found: Type Penalty Overriding Strategic Signal
Before fix: 15 Research actions graded "investigate" by M7 (strategic_score >7.0) were scoring <4.0 because the -1.2 type_boost for Research destroyed the strategic signal.

### Fix: Strategic Override
Added `strategic_override` boolean to `compute_user_priority_score()`:
- When `strategic_score > 7.0`: Research/Thesis type penalty reduced from -1.2 to -0.3
- When `strategic_score <= 7.0`: Full -1.2 penalty preserved
- This preserves the "Research is lower user urgency" signal while respecting M7's assessment of strategic importance

### Key Insight
**Depth grade and user priority score measure different things:**
- Depth grade (M7) = information value / strategic importance
- User priority score (M5) = personal action urgency for Aakash

A Research action can be "investigate" (high strategic value) but still rank lower in user_priority_score because it's agent-delegable. This is correct behavior -- Aakash doesn't need to personally do research; he needs to personally do portfolio check-ins and meetings.

### Remaining Inversions (Acceptable)
- 2 skip actions scoring >7.0: Both Research with high strategic_score + override active. These are strategically important but agent-delegable.
- 14 investigate actions scoring <4.0: Mostly Research with type penalty. Correct for user triage ordering.

## Scoring System Architecture (Final State)

```
Input Factors (7):
  time_sensitivity (w=0.1275)
  conviction_change (w=0.17)
  stakeholder_impact (w=0.17)
  effort_vs_impact (w=0.1275)
  action_novelty (w=0.085)
  irgi_relevance (w=0.17)
  strategic_score (w=0.15)     <-- M7 integration

Boosts:
  type_boost: -1.2 to +1.2 (with strategic override for Research/Thesis)
  priority_boost: -0.4 to +0.8
  source_boost: 0 to +0.6 (Cindy=0.6, ContentAgent=0.2)
  recency_factor: 0 to +0.4
  network_boost: 0 to +1.2 (name match or vector similarity)
  preference_adj: from preference_weight_adjustments table
  staleness_boost: 0 to +0.8 (log curve after 3 days)
  acceptance_rate: multiplicative (type-level accept/dismiss history)

Decay:
  3%/day after 14 days: POWER(0.97, GREATEST(0, days_old - 14))

Normalization:
  Percentile normalization to [1.00-10.00] range
  Run by normalize_all_scores() every 6h via pg_cron

Feedback Loop:
  update_preference_from_rating(action_id, rating 1-5)
  EMA learning (lr=0.3) across 4 dimensions
  Recorded in action_outcomes table
```
