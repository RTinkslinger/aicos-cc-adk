## P0 Bug Fixes (Critical — from M9 cross-audit)

### P0-1: Staleness Boost Was Inverted (FIXED)

**Bug:** `stb := LEAST(0.25 * LN(dp - 2.0), 0.8)` grew logarithmically with age. A 15-day-old action got +0.642 bonus, crowding out fresh P1 items.

**Fix:** Replaced with `stb := 0.5 * EXP(-0.1 * dp)` — exponential decay from +0.50 (fresh) toward 0. Curve: 0-day +0.50, 3-day +0.37, 7-day +0.25, 14-day +0.12, 30-day ~0.02.

**Impact:** Fresh Cindy-Meeting actions (IDs 125-130) went from scores of 1.50-1.99 to 8.10-8.70. Ranking now correctly surfaces recent meeting actions above stale items.

### P0-2: user_triage_queue Was 946ms (FIXED)

**Bug:** View called `compute_user_priority_score(aq.*)` live for every row on every page load — O(n * function_cost) per query.

**Fix:** Replaced live computation with stored `user_priority_score` column reference.

**Performance:** 946ms down to 0.49ms (1930x improvement). The stored column is kept fresh by `normalize_all_scores()` and the 15-min `action_scores` matview cron.

## L61-64: Contextual Scoring — Interaction Recency Boost

**Created:** `interaction_recency_boost(action_row actions_queue)` function

**Design:** Actions from Cindy meetings score higher when the related interaction was recent (<14 days). Links actions to interactions via:
1. Participant name match in action text
2. `linked_people` join to network person names
3. `linked_companies` join to company names

**Boost curve:** `0.8 * EXP(-0.15 * days_since_interaction)` — max +0.80 same-day, decays to 0 by 14 days.

**Integration:** Added as `irb` additive boost in `compute_user_priority_score()`.

**Verified:** All 13 Cindy-Meeting actions get recency boost of 0.78-0.79 (interactions 0-1 days old).

## L65-68: Multi-Objective Ranking

**Created:** `multi_objective_ranking` view + `quick_wins_queue` convenience view

**Three dimensions:**
- **Urgency** (35%): time_sensitivity factor + priority level + age-as-staleness signal
- **Importance** (40%): strategic_score + bucket_impact + IRGI relevance
- **Effort** (25%): effort_vs_impact + action complexity proxy + source reliability

**Composite:** `urgency * 0.35 + importance * 0.40 + effort * 0.25`

**Flags per action:** `quick_win` (high effort + decent importance), `strategic` (importance >= 0.7), `urgent` (urgency >= 0.7)

**Verified:** Portfolio Check-ins rank high (strategic + urgent), fresh Cindy actions surface as quick wins.

**Impact:** Cold-start actions now get reasonable initial differentiation instead of all clustering at the same base score.
