## L21-24: hybrid_search semantic_score Fix

**Bug:** When `query_embedding IS NULL`, `combined_score` was deflated because the semantic weight (0.7) contributed 0 while keyword weight (0.3) capped the maximum. Results appeared to have very low scores.

**Root Cause:** The function used fixed `keyword_weight` and `semantic_weight` regardless of whether an embedding was provided. With NULL embedding, the semantic CTEs returned 0 rows, so combined_score = 0.3 * keyword_RRF + 0.7 * 0 = max ~0.005 instead of ~0.016.

**Fix:** Added `eff_kw_weight` / `eff_sem_weight` variables that normalize weights when `query_embedding IS NULL`:
- NULL embedding: `eff_kw_weight = 1.0, eff_sem_weight = 0.0`
- With embedding: uses original caller weights
- Also added explicit `AND query_embedding IS NOT NULL` guard to all 5 semantic CTEs

**P0 Regression (mid-session):** During initial fix, CTE columns were created without `AS` aliases, causing `r.combined_score does not exist` error. Fixed immediately by restoring all column aliases. M9 flagged the intermediate broken state. Function now fully operational.

**Test Results:**
| Scenario | Before | After |
|----------|--------|-------|
| NULL embedding, top combined_score | 0.0116 | 0.0164 (properly normalized) |
| With embedding, semantic_score | 0.77-1.0 (correct) | 0.77-1.0 (unchanged) |
| With embedding, combined_score | 0.0151 | 0.0164 (correct RRF) |

## L25-28: suggest_actions_for_thesis Diversity Constraint

**Bug:** Same people appeared in network suggestions across most theses. Anupam Rastogi appeared in 6/8 theses, Vedant S in 3/8.

**Root Cause:** The `network_suggestions` CTE ranked by raw similarity only. VC partners with generic "technology/investment" embeddings scored similarly to ALL thesis embeddings.

**Fix:** Added diversity penalty based on how many OTHER theses a person matches:
- `other_count >= 5`: 0.3x score (severe penalty)
- `other_count >= 3`: 0.6x penalty
- `other_count >= 1`: 0.85x penalty
- `other_count = 0`: 1.0x (no penalty, thesis-specific)

Uses HNSW index for initial top-10 candidate selection, then computes diversity penalty on the small set. Pre-fetches thesis embeddings into a CTE to avoid correlated subqueries.

**Test Results:**
| Person | Before (theses) | After (theses) |
|--------|-----------------|----------------|
| Anupam Rastogi | 6/8 | 2/8 |
| Vedant S | 3/8 | 0/8 |
| Max overlap | 6 theses | 2 theses |

## L29-32: USTOL False Positive Elimination

**Bug:** USTOL/Aviation thesis suggestions included irrelevant AI coding content: "How to code with AI agents", "From Writing Code to Managing Agents".

**Root Cause:** `digest_gap_suggestions` used `similarity() > 0.1` (trigram threshold) which let everything through. Vector cosine between USTOL thesis and AI coding digests was 0.82-0.83 -- high because all tech/AI embeddings cluster together.

**Fix:** Replaced `similarity() > 0.1` with:
- Explicit thread name mention in `digest_data` (ILIKE), OR
- Vector cosine similarity > 0.85 (much stricter than 0.1 trigram)

**Test Results:**

| "How to code with AI agents" | Present (false positive) | Removed |
| "From Writing Code to Managing Agents" | Present (false positive) | Removed |
| "Evaluate: Early exploration" | Present (correct) | Present |

`score_action_thesis_relevance` for USTOL was verified clean: only 2 actions score > 0.4 (defense-AI at 0.57, ISRO at 0.54), both legitimately related.

## L33-36: New Intelligence Functions

### find_similar_network(person_id, limit)

Finds network people similar to a given person via 4 methods:
1. **Vector similarity** (primary): cosine distance on embeddings, threshold > 0.5
2. **Shared company connections**: via entity_connections cross-join
3. **Same current company**: array overlap on current_company_ids
4. **Role similarity**: trigram on role_title, threshold > 0.3

Returns: person_id, person_name, similarity_score, match_reason, role_title, shared_context

**Performance:** 104ms (within 200ms target)

### thesis_momentum_report(thesis_id)

Aggregates recent evidence, actions, and signals for momentum scoring:
- Counts recent actions (30-day window) with vector + name matching
- Counts recent digests with vector + name matching
- Counts network connections via entity_connections
- Computes composite momentum_score (0-1.0) from 5 factors:
  - Action density (0.30 weight)
  - Digest density (0.20)
  - Network density (0.20)
  - Evidence depth (0.15)
  - Signal recency (0.15)
- Classifies momentum_direction: accelerating/steady/decelerating/stale/dormant
- Computes signal_velocity (signals per week)
- Returns top_signals JSONB array

**Sample output (Agentic AI Infrastructure):**
- momentum_score: 0.70, direction: "steady"
- 35 recent actions, 17 recent digests, 21 evidence_for, 4 evidence_against
- signal_velocity: 12.09/week

**Performance:** 32ms (within 200ms target)

### Optimization Details

**find_related_entities (435ms -> 85ms, 5.1x):**
- Removed threshold filter from people_by_vector (`> 0.4` prevented HNSW index use)
- Changed to pure `ORDER BY embedding <=> query LIMIT N` which leverages HNSW index
- Removed `NOT IN` subquery (caused sequential scan)

**suggest_actions_for_thesis (631ms -> 25ms, 25.2x):**
- Replaced threshold-based WHERE with `ORDER BY distance LIMIT 10` for HNSW index use
- Reduced candidate evaluation from 2,532 to 10
- Pre-fetched thesis embeddings into CTE for diversity computation
- Diversity penalty computed on 10 candidates * 7 theses = 70 comparisons (was 2,532 * 7 = 17,724)

**aggregate_thesis_evidence (915ms -> 87ms, 10.5x):**
- Split network evidence into two paths: entity_connections (index lookup) + vector (HNSW-limited top-20)
- Raised vector threshold from 0.4 to 0.6 and added LIMIT 20
- Eliminated full-table scan of 3,525 network rows with LEFT JOIN LATERAL
