## Complete IRGI Function Inventory (16 functions)

### Core Search (3)
1. **hybrid_search** — 6-surface RRF search (content_digests, thesis_threads, actions_queue, companies, network, interactions). FTS + vector with configurable weights. Filters: tables, status, date range. **NEW: interactions surface added in L42.**
2. **search_across_surfaces** — Unified FTS search across 6 surfaces returning structured metadata (JSONB). Surface filtering. **NEW in L45.**
3. **find_related_companies** — Vector similarity on company embeddings.

### Entity Intelligence (4)
4. **find_related_entities** — Companies + network people related to a company.
5. **find_similar_network** — 4-method network person similarity (vector, shared company, same company, role trigram).
6. **network_intelligence_report** — Comprehensive person dossier: interactions, obligations, companies, theses, actions, similar people, completeness score. **NEW in L49.**
7. **deal_pipeline_intelligence** — Comprehensive deal dossier: key people, theses, actions, interactions, similar companies, portfolio info, timeline, completeness score. **NEW in L53.**

### Thesis Intelligence (4)
8. **score_action_thesis_relevance** — 4-method weighted thesis-action scoring.
9. **suggest_actions_for_thesis** — Action suggestions with diversity cap (2 per person).
10. **aggregate_thesis_evidence** — Digests + actions + outcomes + network evidence.
11. **detect_thesis_bias** — 6-flag bias detection (confirmation, possible, source, stale, mismatch, thin).
12. **thesis_momentum_report** — Momentum scoring with direction (accelerating/steady/decelerating/stale/dormant).

### Action Intelligence (1)
13. **route_action_to_bucket** — Portfolio/network/thesis routing with network lookup.

### System (3)
14. **update_thesis_bias_flags** — Batch bias flag update on thesis_threads.
15. **interactions_fts_trigger** — Trigger function maintaining FTS on interactions.
16. **irgi_benchmark** — Performance regression test for all 13 intelligence functions. **NEW in L57.**

## Known Limitations

1. **No semantic search without embeddings** — hybrid_search falls back to keyword-only when no query_embedding provided. Edge Function needed for text-to-vector.
2. **network_intelligence_report slow for mega-connected people** — 466+ connections cause connection pool strain. Add LIMIT to company connections CTE.
3. **interactions data is sparse** — Only 11 rows. Intelligence report quality will improve as Cindy processes real email/calendar/WhatsApp data.
4. **deal_pipeline_intelligence portfolio lookup uses ILIKE** — Could break on partial name matches. Should use entity_connections portfolio-company links when available.
