## Loop Results

### L61-64: Search Quality
- **Methodology:** 20 diverse queries tested against `search_across_surfaces` measuring precision@5
- **Before:** 6 GOOD / 3 PARTIAL / 11 MISS (30% precision, FTS-only)
- **After upgrade:** 9 GOOD / 2 PARTIAL / 9 MISS (45% precision)
- **Changes:** Added trigram + ILIKE fallback to `search_across_surfaces`, added portfolio surface
- **Remaining gaps:** 9 queries still miss expected surface. Root cause: FTS tokenization misses non-standard terms ("agentic", "SaaS") and generic terms return digest/company results that outrank the expected surface. Semantic search (embeddings) would resolve, but requires Edge Function for query embedding generation.
- **Search quality score:** 45/100 (FTS-only) -> estimated 80/100 with embeddings

- **Test:** Agentic AI thesis returns 20 connected actions at hop 1
- **Performance:** 2.1ms (PASS)

### L73-76: Smart Suggestions

### L85-88: Benchmark & Optimization
**Critical performance fixes:**

| Function | Before | After | Speedup |

## Known Gaps & Next Steps

1. **Search quality ceiling (45%):** 9 of 20 test queries miss expected surface. Requires Edge Function for query-time embedding generation to enable semantic search via `hybrid_search`. This is the single biggest improvement opportunity.
2. **Competitive landscape similarity:** All competitors show 0.92+ similarity, suggesting embeddings lack differentiation. Will improve as M12 enriches page_content (currently 9.2% coverage).
3. **Interaction data sparse:** Only 11 interactions in system. `interaction_intelligence_report` returns 0 rows for most people. Will improve as M8 Cindy processes real data.
4. **Portfolio surface in search:** Added but `portfolio.fts` may be NULL for many rows -- verify FTS generation trigger exists.
5. **relationship_graph cycle prevention:** Current implementation prevents revisiting seed entity but could still produce loops in complex graphs at depth > 2.
