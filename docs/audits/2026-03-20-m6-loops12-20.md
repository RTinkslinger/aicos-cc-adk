**Analysis:**
- 5/10 queries returned highly relevant top results (AI-related queries work well because the corpus is AI-heavy)
- 3/10 returned irrelevant results ("developer tools" -> Po-Shen Loh, "supply chain" -> Elena Verna, "robotics" -> Olympic Sport)
- 2/10 returned weakly relevant results
- FTS-only mode has poor discrimination for non-AI topics because combined_scores are extremely low (0.006-0.016 range) and barely differentiate
- The function WORKS but quality is degraded without embeddings passed in

**Root cause:** `hybrid_search` requires a pre-computed `query_embedding` vector for semantic search. Without it, only FTS ranking applies, and FTS on generated tsvector columns has limited semantic understanding. The Edge Function (`/search`) is the correct caller -- it generates embeddings via Voyage AI before calling `hybrid_search`.

**Score: 62/100** — FTS-only quality. Estimated 78+ with embeddings (based on company/network similarity quality below). Core queries about AI/thesis topics score well. Non-AI-specific queries degrade significantly.
