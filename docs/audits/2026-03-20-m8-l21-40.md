## Architecture Created

### scripts/granola_sync.py
Reusable CC-to-Postgres bridge for Granola meetings:
- Reads from local cache (`~/Library/Application Support/Granola/cache-v4.json`)
- Or from MCP tool output via stdin
- Supports dry-run, incremental sync, state tracking
- Uses asyncpg for DB writes

### Data Flow (Working)
```
Granola MCP (list_meetings → get_meetings)
    → Signal extraction (summaries, thesis signals, deal signals, obligations)
    → Postgres interactions table
    → people_interactions (identity resolution)
    → obligations (I_OWE / THEY_OWE detection)
    → actions_queue (Cindy-Meeting source)
    → cai_inbox (thesis signals → Megamind, datum_person → Datum)
    → context_gaps (calendar-meeting gap detection)
```
