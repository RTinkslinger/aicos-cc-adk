## L29-32: Search Quality Re-Audit

### hybrid_search Function

**BUG FOUND:** The hybrid_search function has been modified since L20. The current version uses `eff_sem_weight`/`eff_kw_weight` variables and unnamed CTE columns (no column aliases in the SELECT lists of CTEs like `cd_combined`, `tt_combined`, etc.). When the final UNION ALL query references `r.combined_score`, it fails with:

```
ERROR: column r.combined_score does not exist
```

This happens because the CTE columns are unnamed (positional), so `r.combined_score` cannot be resolved.

**Impact:** hybrid_search fails for some weight combinations. The first test (0.5/0.5 weights) succeeded because it hit a different code path (the original function version cached in the connection). Subsequent calls with 1.0/0.0 weights hit the updated function and crashed.

- **What:** Network page_content at 7.6%, email at 0%, e_e_priority at 0.3%

- **What:** 85% of graded actions are "investigate" depth
- **Impact:** Defeats triage purpose -- if everything is investigate, nothing is prioritized
- **Fix:** Tighten auto-grading rules (raise thresholds, add score bands)
