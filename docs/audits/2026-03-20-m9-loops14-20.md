WebFront's Edge Function handles this via the `/search` endpoint, but direct SQL callers cannot use hybrid_search without an embedding.

## Critical Findings

### 1. SCORING COMPRESSION IS THE #1 SYSTEM BOTTLENECK
M5's 4 fixes (LEAST clamp, decay, acceptance, score blend) shifted the compression band upward instead of spreading it. The root cause is insufficient differentiation signals — most actions have similar scoring_factors values (all 0.5 defaults), no network_boost (because e_e_priority is 0.3% populated), and no strategic_score integration. **Fix: integrate strategic_score, populate network priorities, add interaction-based signals.**

### 2. M7 DEPTH GRADING IS DECORATIVE
41 depth grades have been generated but have zero influence on what actions surface to Aakash. The `strategic_score` column on `actions_queue` exists and `rescore_related_actions` writes to it, but `compute_user_priority_score` never reads it. **Fix: add `strategic_score` as a weighted factor in the scoring function.**

### 3. DATA ENRICHMENT HAS STALLED
Every data completeness metric is identical to L13. The M12 agents that were "in progress" (company/network embedding enrichment, web enrichment) appear to not have produced results. The 4 page_content entries and 379 websites are unchanged.

### 4. CIR IS THE BRIGHTEST SPOT
Queue fully drained, 100% fresh state, 7,512 propagation events (up 3,333 from L13), all 8 triggers active. The continuous refresh infrastructure is production-grade.

### 5. NON-COMPANY ENTITIES IN COMPANIES TABLE
Spot check found "Don Bosco School, Kolkata" in the companies table — this is not a company. Data quality cleanup needed for entity type classification.
