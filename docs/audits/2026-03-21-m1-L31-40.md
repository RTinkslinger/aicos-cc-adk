## L31-32: Score Explanation Modal

**What:** Clickable score numbers in action cards now open a detailed explanation modal powered by `explain_score()` Supabase RPC.

**Components:**
- `ExplainScoreModal.tsx` -- bottom sheet (mobile) / centered dialog (desktop)
- `explain-score.ts` -- Server Action wrapper for RPC
- Modified `ActionsBucketList.tsx` to wire clickable scores

**Data flow:** User taps score -> Server Action -> `explain_score(p_action_id)` RPC -> JSONB response with factors, multipliers, formula, percentiles, natural language explanation.

**UI details:**
- 7 factor bars with raw + normalized values, sorted by weight
- Active multiplier badges (type boost, priority, depth, etc.)
- Decay info, percentile rankings, base model breakdown
- Loading skeleton while fetching
- Keyboard Escape to dismiss

**New types:** `ExplainScoreResult`, `ThesisLandscapeRow`, `StrategicBriefingResult`

- 7-step conviction gauge (added Very High level)

**Performance:** N+2 queries reduced to 2 queries (landscape RPC + stats). Significant improvement for 8 theses.

## Risks / Known Issues

- `explain_score()` RPC returns JSONB -- if schema changes, ExplainScoreResult type needs updating
- `thesis_landscape()` returns connected_companies_count which includes broad vector matches (499 for Agentic AI) -- may need capping in display
- Mobile 44px touch target rule excludes small badges via class selector heuristic -- could miss edge cases
- Signal extraction from JSONB is defensive but new signal fields would need manual component updates
