## L41-42: Adversarial Fix (Priority 1)

**Problem:** The thesis detail page's "ADVERSARIAL PERSPECTIVE" section showed generic hardcoded templates from `adversarial-lenses.ts` -- identical boilerplate on every thesis ("Pre-mortem. Assume this fails catastrophically" etc.). User called it useless.

**Fix:**
- Removed imports from `adversarial-lenses.ts` (file still exists for reference but is no longer used by thesis detail page)
- Wired `detect_thesis_bias(thesis_id)` RPC returning 6 real bias flags per thesis:
  - `confirmation_bias` -- evidence ratio extremely skewed
  - `possible_bias` -- ratio elevated but below confirmation threshold
  - `source_bias` -- evidence concentrated from limited sources
  - `stale_evidence` -- recent evidence is thin
  - `thin_evidence` -- total evidence count is low
  - `conviction_mismatch` -- conviction level doesn't align with evidence
- Wired `thesis_health_dashboard()` RPC for per-thesis health vitals:
  - Momentum score and label (HIGH/MEDIUM/LOW)
  - Evidence freshness score and label
  - Health grade (A/A-/B+/B/C/D)
  - Open questions count
  - Evidence ratio (FOR:AGAINST)
  - Network reach, company coverage, recent actions
- If a thesis has no bias flags, shows "No adversarial signals detected" -- not generic templates

**Files changed:**
- `src/app/thesis/[id]/page.tsx` -- replaced section H adversarial framework
- `src/lib/supabase/queries.ts` -- added `fetchThesisBias()`, `fetchThesisHealth()`
- `src/lib/supabase/types.ts` -- added `ThesisBiasResult`, `ThesisBiasFlags`, `ThesisHealthRow`
