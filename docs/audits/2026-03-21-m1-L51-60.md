## L51-52: Fix Tab Home -- Filter Dead Companies

**Problem:** Deadpooled and Exited companies (e.g., "Puch AI") appeared in the Portfolio Health Overview on the home page. Users saw dead companies in the "At Risk" section with Prep buttons, which was misleading.

**Fix:** Added client-side filtering in `src/app/page.tsx`:
- Created `EXCLUDED_STATUSES` set containing "Deadpool" and "Exited"
- Filtered `portfolioCompanies` to exclude those statuses before passing to `PortfolioHealthOverview`
- Recomputed `byHealth` counts from the filtered list so the Green/Yellow/Red numbers are accurate

**Result:**
- Before: 142 portfolio companies shown, including 13 Deadpool + 5 Exited
- After: 124 active companies shown (83 Green, 29 Yellow, 10 Red)
- Attention list now only shows Red companies that are still active (e.g., Navo, CarbonStrong, Mantys) -- no dead companies

**Files changed:** `src/app/page.tsx`

## L53-54: Progressive Disclosure L0/L1

**Problem:** Action cards on `/actions` navigated immediately on click. No way to preview full reasoning, score factors, or source context without leaving the list. Every triage required a round-trip.

**Fix:** Rewrote `src/components/ActionCard.tsx` with two-tier progressive disclosure:
- **L0 (compact):** Priority badge, type, title, thesis tag, date, score, expand arrow
- **L1 (expanded):** Full reasoning under "Why this matters", source context preview, thesis connection link, scoring factors breakdown, meta info (source, created_by), triage/outcome buttons
- **L2 (full detail):** "Full details" link navigates to `/actions/[id]`
- Click toggles expansion instead of navigating
- All triage actions completable from expanded card

**Result:** Users can triage directly from the list without losing context. The list stays compact but reveals depth on demand.

**Files changed:** `src/components/ActionCard.tsx`

## L55-56: Obligation Cards as Stories

**Problem:** Obligation cards showed raw data: "Mohit Gupta, overdue 11 days, introduce, granola". No narrative intelligence -- just a database dump rendered as a card.

**Fix:** Added `buildObligationStory()` and `suggestedActions()` functions to `src/app/comms/page.tsx`:
- Story builder reads obligation `context` JSON (company, amount, deal_stage, targets) and constructs human-readable narratives
- Category-specific templates: introduce, connect, send_document, provide_info, review
- Example output: "You offered to make an introduction for Mohit (Levocred, $2M seed). This has been waiting 16 days."
- Suggested action buttons per card: "Make intro", "Send nudge", "Reschedule", "Not needed", etc.
- Source quotes attributed inline with source name

**Result:** Obligation cards tell stories. Each card is actionable without needing to navigate elsewhere.

**Files changed:** `src/app/comms/page.tsx`
