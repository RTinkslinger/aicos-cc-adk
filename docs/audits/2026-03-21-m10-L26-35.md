## L32-33: Propagation Performance Optimization

### Changes Made
1. **Created `cir_action_entity_map` materialized view** (219 rows: 144 company + 75 network matches)
   - Replaces ILIKE `'%name%'` full table scans in `propagate_company_change()` and `propagate_network_change()`
   - Indexed: unique index + entity lookup index for O(1) queries
   - Refreshed every 15 minutes via cron job #14
   - Graceful fallback: if matview unavailable, falls back to ILIKE

2. **Increased batch size from 200 to 500**
   - Previous: 200 items/min throughput, burst ingest up to 673/min = backlog
   - New: 500 items/min throughput, handles all observed burst patterns

3. **Updated `propagate_company_change()` and `propagate_network_change()`**
   - Use matview join instead of ILIKE scan
