### Gap Analysis: What's Missing for Autonomous Agent Life

**Gap 1: No CIR -> datum_requests auto-trigger**
- When CIR detects a stale entity, it logs it but doesn't create work items for Datum agent
- The `datum_requests` table (0 rows) has the perfect schema: entity_type, entity_id, field_name, source_context, status
- **Fix:** Add INSERT into datum_requests inside the `cir_staleness_refresh()` function when entities exceed staleness thresholds

**Gap 2: No embedding completion notification**
- When network embeddings hit 100%, nothing happens automatically
- The system should auto-run `apply_strategic_recalibration()` and notify
- **Fix:** Add a check to `cleanup-embedding-jobs` cron: if queue empty AND all tables at 100%, fire a cascade event

**Gap 3: No auto-retry for FAILING daily crons**
- The 3 daily crons that failed at 06:00 UTC sit in FAILING state for 24 hours
- `cir-cron-health-check` detects failures but doesn't retry
- **Fix:** Add retry logic to the health check: if a daily job's last run failed > 1 hour ago, re-invoke it

**Gap 4: No content_digests -> ContentAgent trigger**
- When new content_digests are inserted, CIR rescores actions but doesn't signal the ContentAgent
- `cai_inbox` table (1 row) exists for this purpose
- **Fix:** Add a propagation rule: (content_digests, INSERT) -> insert_cai_inbox_message

**Gap 5: No connection pool circuit breaker**
- Connection pool hit 38.3% in v5 during embedding processing
- No automatic throttling when utilization climbs
- **Fix:** Add a check in `process-embeddings`: if pool > 60%, reduce batch_size to 5 and max_requests to 2
