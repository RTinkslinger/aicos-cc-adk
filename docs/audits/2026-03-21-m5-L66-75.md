# M5 Scoring Machine: L66-75 Audit
**Date:** 2026-03-21
**Scope:** Scoring system evolution from SQL-as-intelligence to SQL-as-enabler for agents. New agent-facing APIs, feedback loop, natural language explanations, system context for orchestrator.

---

## Fundamental Shift

Previous loops (L1-65) made the SQL scoring function increasingly sophisticated: 15 multipliers, 7 base factors, z-score normalization, preference learning, score history tracking. **L66-75 changes the paradigm**: the SQL scoring procedure is now a TOOL that agents use as INPUT to their reasoning, not the end of the intelligence chain.

**Before:** Score = final answer. SQL function decides priority.
**After:** Score = starting point. Agents reason about scores with full context and may override, adjust, or explain them.

---

## What Was Built

### L66-67: agent_scoring_context(action_id) -- Score Data API for Agents

Returns **25 top-level JSONB keys** with EVERYTHING an agent needs to reason about an action:

| Section | Content |
|---------|---------|
| Action metadata | id, text, type, priority, status, source, reasoning, thesis_connection, days_old |
| Scores | final_score, confidence, irgi_relevance, strategic_score, base_model, explanation |
| Scoring factors | Raw JSONB from ContentAgent (time_sensitivity, bucket_impact, etc.) |
| Multipliers | All 15 multiplier values (live-computed), combined value |
| Factors normalized | z-score normalized factor values |
| Portfolio context | Company name, ownership_pct, ownership_tier, entry_cheque, valuations, outcomes, health, ops_prio, stage, follow-on decisions, spikey, fumes_date + days_remaining, key_questions, high_impact, external_signal, cash_in_bank, reserves, thesis_connection |
| Network context | person_name, role_title, priority, ryg, linkedin, sourcing_flow, investing_activity |
| Thesis connections | thread_name, core_thesis, conviction, status, key_question_summary, key_companies, investment_implications, evidence previews |
| Related interactions | Last 30 days, via links or text-match fallback. Source, summary, participants, days_ago, deal_signals, thesis_signals, link_type + confidence |
| Obligations | id, person, type, description, status, due_date, is_overdue, cindy_priority, blended_priority |
| Depth grade | auto_depth, approved_depth, reasoning, strategic_score, thesis_connections, diminishing_returns |
| Score history | Last 10 snapshots (score, confidence, computed_at, version) |
| Score trend | trend direction, change_pct, volatility, min/max/avg |
| Agent instructions | Reasoning guidance for the agent |

**Tested:** Actions 118 (Cultured Computers -- pipeline/deals, thesis-connected, interaction-linked) and 120 (AuraML -- portfolio/support, 2.22% ownership, $100K invested). Both return rich, complete context.

### L68-69: enrich_action_context() -- Action Context Enrichment

Enriches `scoring_factors` JSONB so an agent reading it understands the STORY, not just numbers:

| Field Added | Source | Example |
|-------------|--------|---------|
| company_name | Portfolio table match or company_notion_id | "AuraML" |
| company_context | Portfolio attributes | `{ownership_pct: 2.22, health: "Green", stage: "early revenue", follow_on: null, spikey: "0.5", outcome_category: "Cat C SaaS"}` |
| person_name | Network table match | "Vanya" |
| person_priority | Network e_e_priority | "Core" |
| thesis_context | Thesis threads match | "Agentic AI Infrastructure: The harness layer..." |
| source_summary | First 200 chars of source_content | Meeting notes preview |
| context_enriched | Boolean flag | true |
| enriched_at | Timestamp | "2026-03-21 09:43" |

**Coverage:** 49/49 proposed actions enriched (100%), 0 errors. Idempotent via `context_enriched` flag.

### L70-71: Agent Feedback Store + Preference Learning

#### agent_feedback_store TABLE

| Column | Type | Purpose |
|--------|------|---------|
| action_id | int FK | Which action |
| agent_name | text | megamind, eniac, datum, cindy, orchestrator |
| feedback_type | text | score_override, score_adjustment, reasoning, flag, endorsement, context_note |
| feedback_text | text | Agent's reasoning in natural language |
| original_score | numeric | Score at time of feedback |
| suggested_score | numeric | Agent's suggested new score (nullable) |
| adjustment_applied | boolean | Whether this feedback was consumed by preference learning |
| context | jsonb | Additional structured context |

#### Functions

| Function | Purpose |
|----------|---------|
| `record_agent_feedback()` | Agents call this after processing. Validates agent name + type. |
| `agent_feedback_summary()` | Per-action feedback, agent stats, recent 24h, by-type counts |
| `apply_agent_score_overrides()` | Requires 2+ agent consensus (STDDEV < 1.0). Flows into preference_weight_adjustments via EMA (70/30). |

**Self-learning loop:** Agent reasons about action -> writes feedback -> accumulated feedback adjusts scoring weights -> scores improve -> agent reasons better next time.

### L72-73: narrative_score_explanation() -- Natural Language Explanations

**Before (explain_score):**
```
"portfolio_health_multiplier: 1.24, factors: ownership_tier=high"
```

**After (narrative_score_explanation):**
```
"This portfolio/support scores 9.8/10 -- in the top 2% of all actions. AuraML is
a significant position at 2.22% ownership ($100K invested). Linked to thesis:
'AI-Native Enterprise'. Recent interactions add strong intelligence signal (+16%
Cindy boost). Why this ranks high: Deep research depth (grade 3). Megamind rates
this strategically important (8.9/10). Agent feedback: eniac says: 'AuraML
fundraising support should be slightly lower priority...' (suggests 9.0/10)."
```

Narrative includes: opening percentile, portfolio company story (ownership tier, cheque, follow-on, health, fumes), person context, thesis connection with conviction + momentum, obligation pressure, interaction signals, key score drivers, concerns, and recent agent feedback.

## Data Gaps for Future Loops

1. **Key question coverage at 0%** -- 28/142 companies have key_questions (19.7%), but text-match threshold (40% keyword overlap) yields 0 active matches. Needs M12 Data Enrichment.
2. **Compression warning** -- 55.1% of scores in 9-10 bucket. Top-heavy distribution. The multiplicative model with 15 multipliers compounds upward for well-connected actions.
3. **Accept rate only 10.5%** -- 85/95 decided actions dismissed. Preference learning guards prevent activation (needs 100+ decisions, 20%+ accept rate).
4. **Agent feedback just starting** -- 2 feedback records. System is ready for agent-scale usage.
5. **company_notion_id missing on all 49 proposed** -- enrichment relies entirely on text matching. M12 should link actions to companies via notion IDs.
