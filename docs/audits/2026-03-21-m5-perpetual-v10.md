## Critical Bugs Found & Fixed

### Bug 1: normalize_all_scores() fighting refresh_active_scores() (CRITICAL)

**Symptom:** Stored scores drifted 1-3.26 points from live model output despite `refresh_active_scores()` cron running successfully every 30 minutes.

**Root cause:** Two cron jobs conflicted:
- `refresh_active_scores()` (jobid 24, */30 min): computes TRUE model scores, updates stored values
- `normalize_all_scores()` (jobid 6, */6 hours): OVERWRITES model scores with PERCENTILE RANK normalization (prank), spreading scores evenly across 1-10

Every 6 hours, normalize would destroy the model's calibrated scores. Then refresh would detect massive drift and correct them. Then normalize would overwrite again 6 hours later.

**Evidence:**
- Manual `refresh_active_scores()` returned: 26 refreshed, max drift 3.2614
- Cron had run 20 minutes earlier with "succeeded" status
- normalize_all_scores last ran at 18:00 UTC (3.5 hours before investigation)

**Fix:** `SELECT cron.unschedule(6)` — disabled normalize cron permanently. The v5.3 model with 18 multipliers, sigmoid wall at 8.0, and cap [0.4, 1.35] handles distribution without external normalization.

### Bug 2: depth_mult universal 7% penalty (MEDIUM)

**Symptom:** 93.9% of actions had `depth_mult = 0.93` (7% penalty), dragging all scores down.

**Root cause:** `auto_depth` defaults to 1 for all actions. Grade 1 was mapped to 0.93x ("shallowest = penalize"). But grade 1 actually means "uncalibrated" — M6 IRGI hasn't assigned meaningful grades to most actions.

**Fix in compute_user_priority_score():**
- Grade 1: 0.93 → 1.0 (neutral for uncalibrated)
- Grade 0: NEW → 0.90 (explicit shallow penalty when actively downgraded)
- Now uses `approved_depth` when set (user override over auto_depth)

**Impact:** Universal 7% score lift for 93.9% of actions.

### Bug 3: Research type penalty too aggressive (MEDIUM)

**Symptom:** Accepted research actions #113 (3.83) and #14 (3.94) scored far below the acceptance threshold. The user accepted these research actions, proving the model was wrong.

**Root cause:** Research type_mult = 0.82 (18% penalty). Combined with time_decay for older actions and the depth penalty (now fixed), research scores collapsed.

**Fix:** Softened research penalty:
- Primary type_mult: 0.82 → 0.88
- Strategic-high type_mult: 0.92 → 0.95
- Text-fallback type_mult: 0.87 → 0.90
- Text-fallback strategic-high: 0.95 → 0.97
