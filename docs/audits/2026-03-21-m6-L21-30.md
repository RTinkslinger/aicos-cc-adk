## Loop Details

### L21-22: Semantic Score Fix

**Problem diagnosed:** Three interrelated issues in hybrid_search:
1. **Self-match contamination:** Proxy embedding source matched itself with score=1.0
2. **No score differentiation:** RRF rank-only scoring gave all results combined_score 0.008-0.016
3. **AND-only FTS:** `plainto_tsquery` required ALL terms present (missed "pen testing" for "penetration testing")

**Fixes applied:**
- Track proxy source (table + id) and exclude from semantic results via `NOT (proxy_table = 'X' AND id = proxy_id)`
- Replaced combined_score formula: `semantic_weight * cosine_similarity + keyword_weight * RRF_rank` (was `semantic_weight * 1/(60+rank) + keyword_weight * 1/(60+rank)`)
- Added `phraseto_tsquery` OR'd with `plainto_tsquery` for broader FTS matching

**Results:**
| Metric | Before | After |
|--------|--------|-------|
| semantic_score range | 0 for most, 1.0 for self-match | 0.55-0.94, no self-match |
| combined_score range | 0.008-0.016 | 0.49-0.66 |
| Score differentiation | None (all ~0.011) | Meaningful (0.04-0.07 spread) |
