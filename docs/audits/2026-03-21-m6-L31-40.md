## L31-32: Cybersecurity Thesis Search Fix

### Root Cause Found
**P1 bug: proxy self-exclusion in `hybrid_search`**

When searching "cybersecurity", thesis_thread id=1 (Cybersecurity / Pen Testing) became the FTS proxy (highest rank 0.3). The function then **excluded** this record from semantic results via `NOT (proxy_table = 'thesis_threads' AND tt.id = proxy_id)`. The thesis only appeared in keyword results with RRF-based scoring: `0.5 * (1/(60+1))` = **0.008** combined score. Buried at position 8.

### Fix Applied
Changed `hybrid_search` to **include** proxy records with boosted scores:
- Proxy record gets `semantic_score = 1.0` (it IS the query)
- Proxy sorting uses `CASE WHEN proxy THEN 0 ELSE distance END`
- Applied to ALL five table CTEs (content_digests, thesis_threads, actions_queue, companies, network)
- Also changed keyword scoring from RRF (`1/(k+rank)`) to direct `ts_rank_cd` score for better signal

### New Function: `search_thesis_context(query, limit)`
Thesis-focused search combining:
- FTS on thesis fts column
- Trigram name similarity (>0.1 threshold)
- Key questions JSON text search
Returns: thesis_id, thread_name, core_thesis, conviction, status, relevance_score, match_type

### Results
- Cybersecurity thesis: position 8 (0.008) -> **position 1 (0.650)**
- 6/8 thesis names self-match at position 1
- 2 edge cases (Agentic AI Infrastructure, AI-Native Non-Consumption): appear in results but may be outranked by actions with similar semantics
