## L55-56: Search Result Enrichment

### `enriched_search()` — NEW FUNCTION

A wrapper around `hybrid_search()` that adds a `context_snippet` column. For each result, it pulls additional context from the source table:

| Source Table | Context Added |
|-------------|---------------|
| companies | If portfolio: health, ownership%, scale, spikey, key_questions. Otherwise: sector, deal_status, page_content |
| network | role, LinkedIn indicator, RYG status, last_interaction date, page_content |
| actions_queue | priority, action_type, status, thesis_connection, full reasoning |
| thesis_threads | conviction, status, key_companies, evidence_for, evidence_against |
| content_digests | channel, digest_data |

**Design decision:** Built as a wrapper rather than modifying hybrid_search to avoid risk — hybrid_search is the most complex function (~200 lines) and works correctly. The enriched_search function is a pure read-only addition.

**Test result (query: "healthcare AI agents"):**
- Healthcare AI thesis result now shows: `[New] Exploring | Cos: Confido Health (DeVC portfolio, $13M raised, Series A, 10x growth...) | FOR: ++ Confido 10x growth...`
- Actions show full reasoning context
- Was: `Healthcare AI Agents as Infrastructure | core_thesis snippet`

**Latency:** 13.9ms (hybrid_search: ~33ms, wrapper adds minimal overhead due to scalar subqueries on indexed tables) — PASS
