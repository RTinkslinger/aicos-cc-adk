### What's Missing or Wrong

#### BLOCKER 1: `Agent` Tool Not Listed in Capabilities

ENIAC's Section 2 capabilities table lists: Bash, Read, Write, Edit, Grep, Glob, Skill.

**Missing: `Agent`** -- the tool that enables subagent spawning. Without it, ENIAC cannot parallelize research across multiple topics (e.g., researching 3 companies simultaneously for a thesis deep-dive). Content Agent has Agent in its tool list and has 2 defined subagents (web-researcher, content-worker). ENIAC has zero.

Per the persistent agent SDK pattern doc: "Main agents (Cindy, Datum, Megamind, ENIAC): ALL tools allowed."

**Fix:** Add Agent to capabilities + add subagent definitions + add subagent usage guidance to CLAUDE.md.

#### BLOCKER 2: No `.claude/settings.json` (No Hooks)

ENIAC's `.claude/` directory is empty. Every other production agent has:
- `Stop` hook (iteration logging)
- `UserPromptSubmit` hook (manifest check)
- `PreCompact` hook (checkpoint flush)

Without these, ENIAC:
- Won't log iterations to the shared traces file
- Won't checkpoint before compaction (context loss)
- Won't participate in manifest tracking

**Fix:** Create `.claude/settings.json` + hook scripts mirroring Content Agent's pattern.

#### BLOCKER 3: Not in `lifecycle.py`

The lifecycle manager (`orchestrator/lifecycle.py`) creates and manages 4 agents: content, datum, megamind, cindy. ENIAC is completely absent:
- No `ENIAC_WORKSPACE` constant
- No `ENIAC_LIVE_LOG` constant
- No `eniac` in `ClientState` class
- No `build_eniac_options()` function
- No `start_eniac_client()` / `stop_eniac_client()` / `restart_eniac_client()`
- No `send_to_eniac_agent` bridge tool
- No `_read_eniac_response()` background reader
- Not started in `run_agent()` main loop
- Not in `_state_dir()` routing

**Fix:** Add ENIAC as 5th managed agent in lifecycle.py, following the exact same pattern as datum/megamind/cindy.

#### BLOCKER 4: No Orchestrator Routing

HEARTBEAT.md routing table has no `eniac_*` type prefix. There is no way for inbox messages to reach ENIAC. The orchestrator would route research requests to Content Agent instead.

**Fix:** Add `eniac_*` type routing (eniac_research, eniac_thesis, eniac_company, eniac_health_check) to HEARTBEAT.md + Orchestrator CLAUDE.md.

#### BLOCKER 5: No `send_to_eniac_agent` Bridge Tool

Without the bridge tool registered in `create_bridge_server()`, the orchestrator literally cannot communicate with ENIAC. The bridge server currently exposes 4 tools: send_to_content_agent, send_to_datum_agent, send_to_megamind_agent, send_to_cindy_agent.

**Fix:** Add send_to_eniac_agent to bridge server + add to orchestrator's allowed_tools.

## 2. Subagent Architecture Recommendation

ENIAC should use the `Agent` tool to spawn specialized research subagents for parallel work. This follows the Machine > Loop > Step > Army pattern where each step has a serial chain of specialists.

### Recommended Subagents (Filesystem-Based)

Create `.claude/agents/` directory with these agent definitions:

#### `web-researcher.md`
```markdown
---
name: web-researcher
description: Web research specialist for multi-step web investigations
tools:
  - Bash
  - Read
  - Write
  - Skill
  - mcp__web__web_browse
  - mcp__web__web_scrape
  - mcp__web__web_search
  - mcp__web__fingerprint
  - mcp__web__check_strategy
  - mcp__web__manage_session
  - mcp__web__validate
---
You are a web research specialist working for ENIAC (research analyst agent).
Your job: given a research question, use web tools to find, validate, and
synthesize information. Return structured findings with source URLs, dates,
and confidence scores (0.0-1.0). Separate facts from inferences.
```

#### `thesis-analyst.md`
```markdown
---
name: thesis-analyst
description: Thesis analysis specialist for deep thesis health and bias evaluation
tools:
  - Bash
  - Read
  - Skill
  - Grep
---
You are a thesis analyst working for ENIAC. Use psql $DATABASE_URL to query
thesis functions (thesis_health_dashboard, detect_thesis_bias, thesis_momentum_report,
thesis_research_package). Produce structured analysis of thesis health, evidence gaps,
bias indicators, and momentum trends. Load skills/eniac/eniac-thesis-analysis.md for
detailed function signatures.
```

#### `company-profiler.md`
```markdown
---
name: company-profiler
description: Company intelligence specialist for deep company research and profiling
tools:
  - Bash
  - Read
  - Write
  - Skill
  - mcp__web__web_browse
  - mcp__web__web_scrape
  - mcp__web__web_search
---
You are a company intelligence specialist working for ENIAC. Research companies
using web tools and database functions (company_intelligence_profile,
portfolio_deep_context, deal_intelligence_brief). Return structured profiles
with funding, team, market, product, and competitive landscape sections.
Load skills/eniac/eniac-company-intelligence.md for function signatures.
```

### When to Use Subagents

| Scenario | Strategy |
|----------|----------|
| Single thesis deep-dive | Spawn thesis-analyst + 2x web-researcher (for/against evidence) in parallel |
| Company diligence | Spawn company-profiler + web-researcher (competitive landscape) in parallel |
| Health check | Spawn thesis-analyst for thesis health, ENIAC handles system-level signals |
| Queue batch (3+ items) | Process sequentially but spawn subagent for web-heavy research within each item |
| Cross-reference discovery | ENIAC handles DB queries, spawns web-researcher for validation |

### Subagent Anti-Patterns

- Do NOT spawn subagents for simple psql queries -- ENIAC handles those directly
- Do NOT spawn more than 3 subagents simultaneously -- token/memory pressure
- Do NOT let subagents write to the database -- ENIAC calls `eniac_save_research_findings()` after reviewing subagent output
- Do NOT pass raw subagent output to findings -- always synthesize and verify first

### What 2-Hour Autonomy Looks Like

A truly autonomous ENIAC session would:

1. **Receive heartbeat prompt** from orchestrator via bridge tool
2. **Check research queue** -- pick top item
3. **Load research brief** -- understand context
4. **Search existing intelligence** -- avoid redundant work
5. **Spawn subagents** -- parallel web research for evidence gathering
6. **Synthesize findings** -- combine subagent results with DB context
7. **Persist findings** -- `eniac_save_research_findings()` with proper confidence
8. **Cross-reference theses** -- check if findings affect thesis threads
9. **Alert if urgent** -- write to notifications table
10. **Process next queue item** -- loop until queue empty or budget exhausted
11. **Run periodic health checks** -- thesis health, bias, portfolio risk, signals
12. **Handle compaction** -- checkpoint before context wipe, resume after

Steps 1-11 repeat across multiple orchestrator prompts within a session. Step 12 happens automatically via hooks when context gets large.

### Recommended `build_eniac_options()` Specification

```python
def build_eniac_options():
    return ClaudeAgentOptions(
        model=os.environ.get("AGENT_MODEL", "claude-sonnet-4-6"),
        permission_mode="dontAsk",
        allowed_tools=[
            "Bash", "Read", "Write", "Edit", "Grep", "Glob", "Agent", "Skill",
            "mcp__web__web_browse", "mcp__web__web_scrape", "mcp__web__web_search",
            "mcp__web__fingerprint", "mcp__web__check_strategy",
            "mcp__web__manage_session", "mcp__web__validate",
        ],
        mcp_servers={"web": {"type": "http", "url": "http://localhost:8001/mcp"}},
        hooks={"PostToolUse": [HookMatcher(hooks=[eniac_tool_hook])]},
        setting_sources=["project"],
        thinking=ThinkingConfigEnabled(type="enabled", budget_tokens=10000),
        effort="high",
        max_turns=50,       # Research is deep, needs many turns
        max_budget_usd=5.0, # Match Content Agent budget
        cwd=str(ENIAC_WORKSPACE),
        env={
            "ANTHROPIC_API_KEY": os.environ.get("ANTHROPIC_API_KEY", ""),
            "DATABASE_URL": os.environ.get("DATABASE_URL", ""),
            "FIRECRAWL_API_KEY": os.environ.get("FIRECRAWL_API_KEY", ""),
        },
    )
```
