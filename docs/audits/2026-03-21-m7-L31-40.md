### L35-36: Depth Grade Auto-Refresh

**Problem:** No automated system to grade new actions or refresh stale grades. Grading was manual per-session.

**Specialist 1 (Analyst):** All 136 actions graded, 0 stale (all updated within 7 days by prior session). Pyramid shape healthy.

**Specialist 2 (Engineer):** Created `auto_refresh_depth_grades()` function with two steps:
1. **Grade ungraded** -- new actions auto-assigned depth based on status + score + priority + action_type
2. **Refresh stale** -- grades >7 days old recalculated (respects approved_depth, won't override user approvals)

Grading rules:
- Dismissed -> skip(1)
- P0 + score >= 9.5 -> ultra(4)
- P0 + score >= 8.0 + Portfolio/Pipeline -> ultra(4)
- P0 + score >= 8.0 -> investigate(3)
- score >= 7.0 + Portfolio/Pipeline -> investigate(3)
- score >= 4.0 -> scan(2)
- else -> skip(1)

**Specialist 3 (Engineer):** Scheduled cron job 16: `daily_depth_grade_refresh` at 06:30 UTC (after strategic assessment at 06:00).

**Specialist 4 (QA):** Ran function -- returned empty (all current, no stale). Verified pyramid:

| skip | 53 | 36.8% | ~35% |
| scan | 46 | 31.9% | ~35% |

All within 3% of targets. Pyramid health: **healthy**.
