### L61-62: `megamind_agent_context()` -- NEW

| Section | Contents | Data Source |
|---------|----------|-------------|
| `portfolio_risk` | CRITICAL/HIGH companies + portfolio summary | `portfolio_risk_assessment()` + portfolio table |

| `obligations` | Health summary, I-owe/they-owe urgent items with details | obligations table |
| `recent_cascades` | Last 7 days of cascade events with summaries | cascade_events table |
| `thesis_momentum` | Per-thesis conviction, momentum tier, portfolio exposure | thesis_threads + portfolio + actions_queue |
| `strategic_network` | Top 15 people by importance with narratives | `strategic_network_map()` |
| `pending_decisions` | Top 10 actions needing decision | `actions_needing_decision()` |
| `recent_interactions` | Last 30 days of interactions with signals | interactions table |
| `system_capabilities` | What the system can do (booleans) | Static |

**Verification:**

### L63-64: `actions_needing_decision_v2(p_limit)` -- NEW
**Purpose:** Enhanced decision queue with full context per action. An agent reading this can write a chief-of-staff recommendation without querying anything else.

**Returns per action (beyond v1):**
| Column | Contents |
|--------|----------|
| `company_context` | Full portfolio company data: health, financials, key questions, follow-on, external signals, other open actions |
| `person_context` | People connected to the company via entity_connections: interaction recency, obligation counts |
| `thesis_context` | Related thesis threads: conviction, momentum tier, bias severity |
| `obligation_context` | Obligations linked to this action or matching by description |
| `interaction_signals` | Recent interactions relevant to this action (by company or thesis) |
| `recommendation` | Structured reasoning: STALE/OBLIGATION/RED COMPANY/HIGH SCORE/LOW SCORE/Standard |

**Verification:**
- 15/15 actions returned with recommendations
- 0/15 with company_context (company_notion_id not populated on most actions -- M12 data enrichment dependency)
- 1/15 with thesis context, 1/15 with interaction signals
- All 15 with recommendation text

**Note:** The low context match rate is a data quality issue, not a function issue. Once M12 Data Enrichment populates `company_notion_id` on actions_queue rows, this function will return rich company context. The function correctly returns NULL when no match exists.

## Architecture Notes

### Design Principle
Every function returns rich, structured JSONB that an agent can reason about without needing to make additional queries. The SQL layer is plumbing; the agent layer is intelligence.

### Dependency Graph
```
megamind_system_report() v5.0
  -> megamind_agent_context()
       -> portfolio_risk_assessment()
       -> actions_needing_decision()
       -> strategic_network_map()
  -> function_catalog (information_schema)

generate_strategic_narrative('upcoming_decisions')
  -> actions_needing_decision_v2()
       -> portfolio table (company_context)
       -> entity_connections + network (person_context)
       -> thesis_threads (thesis_context)
       -> obligations (obligation_context)
       -> interactions (interaction_signals)

cascade_impact_analysis()
  -> cascade_events
  -> depth_grades (downstream changes)
  -> obligations (downstream changes)
  -> thesis_threads (blast radius details)
  -> portfolio (blast radius details)
```

### M12 Dependency
The `actions_needing_decision_v2()` function's company_context, person_context, and interaction_signals are all gated by `company_notion_id` being populated on `actions_queue` rows. M12 Data Enrichment should prioritize linking actions to portfolio companies to unlock the full power of these agent-ready functions.
