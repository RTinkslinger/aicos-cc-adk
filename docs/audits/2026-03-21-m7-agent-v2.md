## Problem Statement

Megamind's CLAUDE.md was a 1,137-line **scripted processing manual** — step-by-step algorithms for depth grading (11 steps), cascade processing (10 steps), and strategic assessment (10 steps), plus ~150 lines of raw SQL templates and duplicated domain knowledge already present in skill files.

This violates the persistent ClaudeSDKClient pattern: "CLAUDE.md = Objectives, Not Scripts."

A persistent agent running as ClaudeSDKClient reasons autonomously about tool chaining. Step-by-step scripts disable the agent's ability to compose operations creatively, and they duplicate knowledge that belongs in skills (loaded on demand, not always in context).

## What Was Done

### Full Rewrite: Objectives + Identity + Boundaries

The CLAUDE.md was rewritten from scratch following the persistent agent SDK pattern:

| Section | Lines | Purpose |
|---------|-------|---------|
| Identity | 25 | Who Megamind IS and IS NOT. Co-strategist framing. |
| Objectives (4) | 40 | Standing objectives: ROI optimization, cascade processing, strategic health, strategic sparring |
| Priority Buckets | 12 | The 4 time-allocation buckets (reference table) |
| Tools & Capabilities | 45 | ALL tools enabled. DB access method. Tables read/write/never. Column gotchas. |
| Skills Reference | 30 | Skill file index + loading matrix |
| SQL Functions Inventory | 80 | All 45 functions organized by category (kept — this IS the agent's power tool reference) |
| Convergence Rules | 15 | 6 hard constraints, each in 1-2 lines |
| Trust Ramp | 10 | 3-level table + check command |
| Collaboration Model | 20 | Who reads from whom (kept — necessary for correct data routing) |
| State Tracking | 15 | State files + checkpoint protocol |
| ACK Protocol | 10 | Required output format |
| Boundaries | 30 | 18 "never" rules |
| **Total** | **342** | **70% reduction from 1,137** |

### Key Design Decisions

**1. Objectives replace algorithms.**

Before: "Step 1: Read the action(s). Step 2: Load thesis thread context. Step 3: Load recent depth_grades..."

After: "Grade agent-delegated actions for depth. Compute strategic ROI using the 5-component formula (load skills/megamind/strategic-reasoning.md)."

The agent loads the skill file when it needs the formula. It reasons about which steps to take. It doesn't follow a numbered recipe.

**2. SQL functions inventory stays.**

The 45-function inventory was kept (80 lines). This is NOT a script — it's a tool catalog. The agent needs to know what functions exist so it can call them. This is the equivalent of a tool manifest. Without it, the agent would write raw queries from scratch (the exact problem the previous audit identified).

**3. Tool restrictions removed.**

v1 said "No web tools. No direct content analysis." The persistent pattern says ALL tools allowed. Megamind may need web tools for strategic research, may need Agent tool for subagent delegation. Artificial restrictions reduce capability.

**4. Orchestrator dependency softened.**

v1 was written as if Megamind passively waits for Orchestrator prompts. v2 frames objectives as standing — "pursue autonomously whenever you have relevant context." The Orchestrator still sends heartbeats, but Megamind reasons about what to do, not just follows instructions.

**5. Column name gotchas preserved.**

The `action` vs `action_text` and pipe-delimited vs array differences are genuine traps. These 8 lines prevent costly runtime errors. Kept.

**6. ACK protocol preserved.**

The Orchestrator parses ACK output. This is an interface contract, not a script. Kept as-is.
