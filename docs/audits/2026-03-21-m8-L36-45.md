## L38-39: Interaction-Action Linkage

### Problem
- Zero explicit links between interactions and actions
- M5 scoring's `interaction_recency_boost()` relied on text matching only
- Only ~3 actions received any interaction boost

### Solution
1. **Created `interaction_action_links` table** with unique(interaction_id, action_id)
2. **Created `link_interactions_to_actions()` function** with 4 matching strategies:
   - Company name match (deal_company in action text): 0.85 confidence
   - Person name match (linked person appears in action): 0.80
   - Linked company name match: 0.80
   - Source topic match: 0.70
3. **Enhanced `interaction_recency_boost()`** with Strategy 0: direct table lookup via `interaction_action_links`

### Results
| Metric | Before | After |
|--------|--------|-------|
| interaction-action links | 0 | **31** |
| Interactions with action links | 0 | **15** (65.2%) |
| Actions with interaction links | ~3 | **24** (17.6% of 136) |
| Boost range | 0-0.2 | **0.12-0.79** |

## L40-41: Obligation Tracking Enhancement

### Problem
- 38 obligations, 14 overdue, 2 escalated
- 7 overdue obligations linked to "Dismissed" actions — no active follow-up
- No health summary function for monitoring

### Solution
1. **Created `obligation_health_summary()` function** returning JSONB:
   - Counts: total, overdue, urgent, upcoming, fulfilled
   - Metrics: avg_days_overdue, i_owe/they_owe breakdown
   - Health score (1-10 scale based on overdue count)
   - Top 5 most critical with person context

2. **Created `auto_generate_obligation_followup_actions()` function**:
   - Finds overdue obligations whose linked action was "Dismissed"
   - Auto-generates follow-up actions with "OVERDUE (Nd):" prefix
   - Updates obligation to link to new action
   - Idempotent (won't create duplicates)

3. **Generated 8 follow-up actions** for dismissed-action obligations
