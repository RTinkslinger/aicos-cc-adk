## TOP 5 ISSUES BLOCKING USER VALUE

1. **Action-company linkage = 0%** — Actions float without company context. Portfolio-centric intelligence is impossible. The scoring_factors JSONB has company context for some actions but the structured `company_notion_id` column is empty. This is the single highest-impact fix.

2. **Network embeddings 85% missing** — 3,006 people invisible to semantic search. Queue stalled with connection failures. Until network has embeddings, "who should I talk to about X?" returns garbage for most of the network.

3. **key_questions 82% empty** — The most actionable field for portfolio check-ins. M12 doubled it (14->28) but 114 companies still have nothing. No agenda = no intelligence for meetings.

4. **23 intelligence-in-SQL functions** — The system tries to generate strategic assessments, predict actions, detect opportunities, and compose briefings IN SQL. These should be agent reasoning. The SQL should provide context; the agent should think.

5. **Content pipeline stalled** — No new digests in 5 days. The content-to-intelligence loop is broken.

## WHAT WOULD MOVE THE NEEDLE TO 5/10

1. Fix `company_notion_id` on actions_queue — one SQL UPDATE correlating action text/scoring_factors to companies. Could linkage from 0% to 60%+ in one operation.
2. Fix embedding Edge Function connection failures — the queue has the work, but connections keep failing. May need a connection pool fix or Edge Function retry logic.
3. M12 to push key_questions to 50%+ (71 more companies). Use the same enrichment pattern that produced the good ones.
4. Resume content pipeline — 5 days without new digests means new content isn't becoming intelligence.
5. One agent-produced intelligence artifact per portfolio company — even a single sentence of strategic assessment stored in a `strategic_notes` column would beat the current "field concatenation" approach.
