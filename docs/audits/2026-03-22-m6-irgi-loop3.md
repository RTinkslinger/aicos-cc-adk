## Priority 1: interaction_thesis_xref Performance Fix — DONE

**Problem:** 1006ms execution time (was 73ms in loop 2 when interactions=23, grew to 334 interactions). The CROSS JOIN between 334 interactions x 8 theses = 2,672 rows, each running `regexp_split_to_table` for key question word matching — O(n^2) text splitting per row.

**Root Cause:** The function used `regexp_split_to_table(lower(summary), '\W+')` nested inside `regexp_split_to_table(lower(kq_text), '\W+')` inside `EXISTS` — creating a quadratic word-by-word comparison for every cross-join row. As interactions grew 14.5x (23 to 334), latency grew super-linearly.

**Fix:** Two-part rewrite:
1. **LATERAL JOIN with ORDER BY** instead of CROSS JOIN + filter: `ORDER BY i.embedding <=> td.embedding LIMIT 30` per thesis leverages the IVFFlat index instead of computing all 2,672 distance pairs
2. **FTS tsvector overlap** instead of regexp word splitting: Pre-compute `to_tsvector('english', string_agg(kq_text))` per thesis once, then check `interaction_fts @@ plainto_tsquery(kq_text)` — uses the existing GIN index on interactions.fts

**Result:**
| Metric | Before | After | Improvement |
|--------|--------|-------|-------------|
| Cold execution | 1006ms | 152ms | 6.6x |
| Warm execution | 1006ms | 17ms | 58x |
| Cross-join rows | 2,672 | 240 (8 x 30) | 11x fewer |
| Index usage | None (seq scan) | IVFFlat + GIN | Yes |
| Results quality | 30 rows, 7 theses | 30 rows, 6 theses | Equivalent |

## Priority 3: WhatsApp Search Improvements — DONE

**3a. IVFFlat Index Rebuild:**
- Old: 10 lists for 637 embedded rows (suboptimal — should be sqrt(n))
- New: 25 lists for 713 embedded rows
- Impact: Better recall and faster ANN queries

**3b. Trigram Index on chat_name:**
- Added `idx_whatsapp_chat_name_trgm` (gin_trgm_ops)
- Accelerates ILIKE queries in `person_holistic_search` and `company_holistic_search` LATERAL joins

**3c. WhatsApp Enrichment in agent_search_context:**
- Added `WHEN bs.source_table = 'whatsapp'` handling to all 7 enrichment columns:
  - `why_it_matters`: chat type, message count, participant count, last active date with recency label, group_context, summary
  - `portfolio_connection`: cross-checks chat_name against portfolio company names
  - `thesis_relevance`: FTS match of chat content against thesis thread names
  - `recent_signals`: conversation summary
  - `interaction_recency`: last message date with activity status
  - `agent_action_hint`: contextual hints (DEAL INTEL for DeVC groups, RELATIONSHIP for high-volume 1:1, ACTIVE for recent, CONTEXT for older)

**Test Result:** Query "Rajat Agarwal WhatsApp messages" now returns WhatsApp results with full enrichment context (was returning NULLs for all enrichment columns before).
