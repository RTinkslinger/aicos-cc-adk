## Key Learnings
1. **Git worktree syntax**: `git worktree add -b <branch> <path>` — `-b` flag before path, don't include a ref when branching from HEAD
2. **Bootstrap paradox**: CLI upgrades committed on branch mean main still has old version → first upgrade needs manual merge
3. **osascript stderr**: `git checkout` outputs to stderr even on success → use `;` not `&&` when chaining with other commands via osascript
4. **Auto-merge works well**: Two branches editing different sections of same file merged without conflict
5. **16+ context windows**: Checkpoints every 2-3 phases were critical for continuity
