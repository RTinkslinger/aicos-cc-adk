# M7 Megamind — Machine Context
*Accumulated across loops. Product leadership reads at start, writes at end.*

## Product Vision Context
Megamind is Aakash's co-strategist -- an equal partner, not a servant. He reasons about ROI, opportunity cost, diminishing returns, and convergence. He pushes back when priorities are wrong, surfaces contradictions and blind spots, and ensures the action queue converges rather than endlessly expanding.

User cares about: correct prioritization of obligations and actions, honest strategic assessment (no inflated scores), morning context that tells a story ("your key decision today is X" not "here are 146 actions"), and convergence -- the system should resolve more than it generates.

**Megamind is a persistent Claude Agent SDK agent on the droplet.** He REASONS about strategy, priorities, contradictions, and morning context. His intelligence outputs are WRITTEN to DB tables. WebFront READS those tables. SQL functions that generate briefings, detect contradictions, or compute strategic priorities are WRONG -- those are agent outputs, not SQL computations.

Megamind's output feeds: M1 WebFront (reads agent-written strategic briefing, morning context, contradiction alerts from DB), M5 Scoring (score_vs_user_decision contradictions for calibration), M8 Cindy (obligation priority blending at 40% Megamind weight), and ENIAC (action routing with calibrated execution prompts).

## Architecture Overhaul

### Current State (WRONG -- SQL generates intelligence)
~22 SQL functions, many of which encode intelligence logic that belongs in the agent:

**MUST MIGRATE TO AGENT OUTPUTS (SQL generates intelligence -- violation):**
- `format_strategic_briefing()` -- SQL assembles narrative briefing. Should be agent-written output stored in DB.
- `megamind_morning_strategic_context()` -- SQL computes headline, todays_key_decision, strategic_risks, contradictions_summary. This is REASONING, not data access. Agent writes this.
- `megamind_strategic_contradiction_detector()` -- SQL runs 6 detection types (health_vs_followon, score_vs_user_decision, etc.). Pattern detection = agent reasoning, not SQL CASE statements.
- `megamind_daily_priorities()` -- SQL computes your_focus_today, agent_executing, portfolio_alerts. Prioritization = agent reasoning.
- `megamind_action_routing()` -- SQL routes actions to HUMAN_DECISION / AGENT_EXECUTE / AGENT_PREPARE. Routing decisions = agent reasoning.
- `megamind_honest_scorecard()` -- SQL self-assessment. Agent should write its own assessment.

**STAYS AS SQL (utility tools the agent calls):**
- `get_convergence_ratio()` -- simple COUNT of open vs resolved. Deterministic formula.
- `megamind_score_obligations()` -- HYBRID. SQL executes the scoring formula (deterministic computation), but agent designs and can override the formula with contextual reasoning. Agent can say "Surabhi should be higher because of deal timing" and write an override.
- `megamind_route_to_agent()` -- simple UPDATE that sets status + depth_grade. The routing DECISION is agent reasoning; the write is plumbing.
- `agent_search_context()`, `balanced_search()` -- search/retrieval tools.
- Thread RPC functions (`megamind_task_create`, `megamind_task_list`, `megamind_task_thread`, `megamind_task_respond`, `megamind_task_auto_process`, `megamind_task_post_message`) -- plumbing for agent communication.

### Target State (agent-output architecture)
1. **Megamind agent runs on droplet** (persistent ClaudeSDKClient via lifecycle.py).
2. **Agent REASONS** about strategy, priorities, contradictions, morning context.
3. **Agent WRITES outputs** to DB tables (e.g., `agent_outputs`, `megamind_briefings`, or structured columns in existing tables).
4. **WebFront READS** those agent-written outputs. No SQL function generates intelligence at render time.
5. **Scoring is HYBRID:** SQL formula for deterministic computation (fast, consistent), agent designs the formula and can override individual scores with contextual reasoning. Agent writes overrides to DB; scoring function checks for overrides before applying formula.

### Migration Path
- Phase 1: Agent writes morning context + strategic briefing to a `megamind_outputs` table. WebFront reads from that table instead of calling `megamind_morning_strategic_context()`.
- Phase 2: Agent runs contradiction detection as part of its reasoning loop, writes findings to DB. WebFront reads stored contradictions instead of calling `megamind_strategic_contradiction_detector()`.
- Phase 3: Agent writes daily priorities and action routing decisions to DB. SQL routing function becomes a simple "apply what agent decided" write.
- Phase 4: Deprecate intelligence-generating SQL functions. Keep utility tools only.

## Patterns of Success
- **Strategic recalibration works.** M7-to-M5 feedback loop wired via cron: contradiction findings feed scoring calibration. score_vs_user_decision contradictions are the purest signal for model improvement.
- **Route research to ENIAC, not human.** Low-ROI thesis research can be auto-delegated, freeing user attention for portfolio/deal decisions.
- **Morning context as narrative.** "Your key decision today is Cultured Computers $150-300K" works better than "23 actions need attention." The agent should WRITE this narrative, not SQL.
- **Obligation scoring with 5-factor formula (HYBRID).** SQL executes the formula for speed and consistency. Agent designs the formula and overrides individual scores when context demands it. Creates real differentiation (range 0.245-0.773 vs all-1.0 before).
- **Contradiction detection surfaces real gaps.** 30 high_ownership_silent contradictions = real signal that WhatsApp interactions are not linked to portfolio companies. This detection is agent REASONING, not SQL pattern matching.

## Anti-Patterns
- **SQL generating intelligence.** ~10 of 22 SQL functions generate briefings, detect contradictions, compute priorities. This is agent work. SQL should fetch data; agent should reason about it and write outputs.
- **Convergence inflation.** Counting Accepted as resolved inflated the ratio. Accepted = user validated but not done. Only Dismissed/Done/expired are resolved.
- **Score scale corruption.** `strategic_score` column is 0.0-1.0 numeric. Previous writes put 0-10 values. Always multiply by 10 for display, never store display values.
- **Self-grading inflation.** Machine reported 9.6/10 while user experienced 3/10. Technical health (crons passing, embeddings 100%) is not product quality. Only the user judges.
- **Generic auto_process without entity extraction.** "should we follow on AuraML?" must scope to AuraML, not return a generic portfolio risk assessment.
- **WebFront calling SQL functions that generate intelligence at render time.** WebFront should READ pre-computed agent outputs from DB, not trigger SQL computation. Agent runs on its own schedule and writes outputs; WebFront reads them.
