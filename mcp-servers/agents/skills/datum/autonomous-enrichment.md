# Datum Autonomous Enrichment Skill

Ongoing data enrichment patterns that Datum runs as background maintenance.
This skill codifies M12 Data Enrichment Machine loops into reusable Datum operations.
Load this skill when running scheduled enrichment, processing new entities from M4,
or when data quality metrics show regression.

---

## Overview

M12 Data Enrichment ran 15+ loops and achieved baseline coverage. Datum now owns
continuous enrichment as an ongoing responsibility. The patterns below are what
M12 discovered works -- Datum applies them autonomously.

**Key principle:** Enrichment is maintenance, not a project. New entities arrive
daily from content pipeline, meetings, and manual additions. Each needs the same
enrichment treatment M12 gave the initial corpus.

## 1. New Entity Enrichment (Trigger: entity creation)

When Datum creates or receives a new entity, apply this enrichment checklist:

### For Companies

```sql
-- Step 1: Generate page_content from structured fields
UPDATE companies SET
  page_content = CONCAT(
    name, '. ',
    COALESCE(type || '. ', ''),
    COALESCE('Sector: ' || sector || '. ', ''),
    COALESCE('Deal Status: ' || deal_status || '. ', ''),
    COALESCE('Pipeline: ' || pipeline_status || '. ', ''),
    COALESCE('Type: ' || type || '. ', ''),
    COALESCE('Sells to: ' || array_to_string(sells_to, ', ') || '. ', ''),
    COALESCE('Website: ' || website || '. ', ''),
    COALESCE('Domain: ' || domain || '. ', ''),
    COALESCE('Funding: ' || venture_funding || '. ', ''),
    COALESCE('Founded: ' || founding_timeline || '.', '')
  ),
  enrichment_status = 'datum-enriched',
  last_enriched_at = NOW()
WHERE id = $company_id
  AND (page_content IS NULL OR page_content = '');
```

**Sector classification rules (12-phase):**
1. Name contains university/college/institute/school -> sector = 'Education'
2. Name/type contains 'venture'/'capital'/'fund'/'investment' -> sector = 'Venture Capital Firm'
3. Keywords fintech/insurance/banking/payments -> sector = 'Financial Services'
4. Keywords SaaS/API/developer/platform + sells_to developers/enterprise -> sector = 'SaaS'
5. Keywords D2C/consumer/app/social + sells_to consumers -> sector = 'Consumer'
6. Keywords logistics/supply chain/manufacturing -> sector = 'B2B'
7. Keywords AI/ML/biotech/cleantech/quantum -> sector = 'Frontier'
8. Known company names from general knowledge
9. Unnamed/phantom entries -> sector = 'Other'
10. All remaining unclassifiable -> sector = 'Other'

**Domain generation (if website missing but name is clean):**
```sql
-- Only for companies with clean names (3-30 chars, not stealth/unnamed)
UPDATE companies SET
  website = 'https://' || LOWER(REGEXP_REPLACE(name, '[^a-zA-Z0-9]', '', 'g')) || '.com',
  domain = LOWER(REGEXP_REPLACE(name, '[^a-zA-Z0-9]', '', 'g')) || '.com'
WHERE id = $company_id
  AND website IS NULL
  AND length(name) BETWEEN 3 AND 30
  AND name NOT ILIKE '%stealth%'
  AND name NOT ILIKE '%unnamed%'
  AND name NOT ILIKE '%university%';
```
**Note:** Generated domains have low confidence. Mark with `enrichment_source = 'domain_generated'`.

### For Network Members

```sql
-- Step 1: Ensure page_content exists
-- page_content for network is auto-generated by triggers, but verify
SELECT id, person_name, length(page_content) FROM network WHERE id = $person_id;

-- Step 2: Generate email from name + company domain (confidence 0.4)
-- Only if: person has company link, company has domain, person has no email
INSERT INTO identity_map (entity_type, entity_id, identifier_type, identifier_value,
                          confidence, source, created_at)
SELECT 'network', n.id, 'email',
  LOWER(split_part(n.person_name, ' ', 1)) || '.' ||
  LOWER(split_part(n.person_name, ' ', 2)) || '@' || c.domain,
  0.4, 'pattern_generated', NOW()
FROM network n
JOIN entity_connections ec ON ec.source_type = 'network' AND ec.source_id = n.id
  AND ec.connection_type = 'current_employee'
JOIN companies c ON c.id = ec.target_id AND c.domain IS NOT NULL
WHERE n.id = $person_id
  AND n.email IS NULL
  AND NOT EXISTS (SELECT 1 FROM identity_map WHERE entity_id = n.id AND identifier_type = 'email');
```

## 2. Batch Dedup Sweep (Trigger: weekly or post-batch-import)

YC batch imports and content pipeline often create duplicates with batch suffixes.

### Auto-Merge Pattern: Batch Suffix Duplicates

These are high-confidence duplicates where one name is `X` and the other is `X (YC W26)`:

```sql
-- Find batch suffix duplicates
SELECT a.id as winner_id, b.id as loser_id, a.name as winner_name, b.name as loser_name
FROM companies a
JOIN companies b ON b.name ~ ('^' || regexp_replace(a.name, '([.+*?^${}()|[\]\\])', '\\\1', 'g') || ' \(')
WHERE a.id < b.id
  AND b.pipeline_status IN ('NA', '')
  AND a.pipeline_status NOT IN ('Merged/Duplicate');
```

**Merge steps (always in this order):**
1. Move entity_connections from loser to winner (skip if duplicate link exists)
2. Delete orphaned entity_connections still pointing to loser
3. COALESCE data from loser into winner (never overwrite non-NULL)
4. Mark loser as `pipeline_status = 'Merged/Duplicate'`

### Non-Batch Duplicates (medium confidence)

For matches like "Ola Cabs" vs "Ola" or "KPMG UK" vs "KPMG" -- these are
legitimately different entities in many cases. Create datum_requests, do NOT auto-merge.

## 3. Portfolio Enrichment (Trigger: new portfolio entry or Notion sync)

Portfolio page_content must be rich (300+ chars) for search and scoring.

```sql
-- Generate rich page_content from 14 structured fields
UPDATE portfolio SET
  page_content = CONCAT(
    portfolio_co, '. ',
    COALESCE('Health: ' || health || '. ', ''),
    COALESCE('Stage: ' || current_stage || '. ', ''),
    COALESCE('Entry cheque: $' || entry_cheque::text || '. ', ''),
    COALESCE('Ownership: ' || (ownership_pct * 100)::text || '%. ', ''),
    COALESCE('Entry round valuation: $' || entry_round_valuation::text || '. ', ''),
    COALESCE('Last round valuation: $' || last_round_valuation::text || '. ', ''),
    COALESCE('Follow-on decision: ' || follow_on_decision || '. ', ''),
    COALESCE('Scale: ' || scale_of_business || '. ', ''),
    COALESCE('Thesis: ' || thesis_connection || '. ', ''),
    COALESCE('Key questions: ' || key_questions || '. ', ''),
    COALESCE('High impact: ' || high_impact || '. ', ''),
    COALESCE('External signal: ' || external_signal || '. ', ''),
    COALESCE('Outcome: ' || outcome_category || '.', '')
  ),
  updated_at = NOW()
WHERE id = $portfolio_id
  AND (page_content IS NULL OR length(page_content) < 100);
```

## 4. Embedding Queue Monitoring

After any batch enrichment, check the embedding queue:

```sql
-- Check queue depth
SELECT count(*) as pending FROM pgmq.q_embedding_jobs WHERE read_ct = 0;

-- Check for stuck jobs (read but not processed)
SELECT count(*) as stuck FROM pgmq.q_embedding_jobs WHERE read_ct > 3;

-- Verify coverage after queue drains
SELECT 'companies' as entity,
  count(*) - count(embedding) as missing
FROM companies
UNION ALL
SELECT 'network', count(*) - count(embedding) FROM network
UNION ALL
SELECT 'portfolio', count(*) - count(embedding) FROM portfolio;
```

If stuck > 0, the embedding cron job may have failed. Log a notification.

## 5. Cross-Machine Regression Detection

Other machines (M4 Datum, M5 Scoring, M10 CIR) modify entity data. Check for regressions:

```sql
-- Companies with empty page_content (should be 0)
SELECT id, name, enrichment_status, datum_source
FROM companies
WHERE page_content IS NULL OR page_content = ''
ORDER BY created_at DESC;

-- Companies with null sector (should be 0)
SELECT id, name, enrichment_status
FROM companies
WHERE sector IS NULL OR sector = ''
ORDER BY created_at DESC;

-- Network with empty page_content (should be 0)
SELECT id, person_name, enrichment_status
FROM network
WHERE page_content IS NULL OR page_content = ''
ORDER BY created_at DESC;
```

If any rows returned: fix immediately using the enrichment patterns above.
Log: "M12 regression fix: N records had empty page_content from [source], fixed."

## 6. Data Quality Scorecard

Run this composite query to get the full enrichment health scorecard:

```sql
SELECT
  -- Companies
  'companies_total' as metric, count(*)::text as value FROM companies
UNION ALL SELECT 'companies_empty_content', count(*)::text FROM companies WHERE page_content IS NULL OR page_content = ''
UNION ALL SELECT 'companies_thin_content', count(*)::text FROM companies WHERE length(page_content) < 50
UNION ALL SELECT 'companies_null_sector', count(*)::text FROM companies WHERE sector IS NULL OR sector = ''
UNION ALL SELECT 'companies_has_website', count(*)::text FROM companies WHERE website IS NOT NULL AND website != ''
UNION ALL SELECT 'companies_has_embedding', count(*)::text FROM companies WHERE embedding IS NOT NULL
UNION ALL SELECT 'companies_duplicates', count(*)::text FROM companies WHERE pipeline_status = 'Merged/Duplicate'
-- Network
UNION ALL SELECT 'network_total', count(*)::text FROM network
UNION ALL SELECT 'network_empty_content', count(*)::text FROM network WHERE page_content IS NULL OR page_content = ''
UNION ALL SELECT 'network_has_email', count(*)::text FROM network WHERE email IS NOT NULL AND email != ''
UNION ALL SELECT 'network_has_linkedin', count(*)::text FROM network WHERE linkedin IS NOT NULL AND linkedin != ''
UNION ALL SELECT 'network_has_embedding', count(*)::text FROM network WHERE embedding IS NOT NULL
-- Portfolio
UNION ALL SELECT 'portfolio_total', count(*)::text FROM portfolio
UNION ALL SELECT 'portfolio_empty_content', count(*)::text FROM portfolio WHERE page_content IS NULL OR page_content = ''
UNION ALL SELECT 'portfolio_has_thesis', count(*)::text FROM portfolio WHERE thesis_connection IS NOT NULL AND thesis_connection != ''
UNION ALL SELECT 'portfolio_has_embedding', count(*)::text FROM portfolio WHERE embedding IS NOT NULL
-- Cross-entity
UNION ALL SELECT 'entity_connections', count(*)::text FROM entity_connections
UNION ALL SELECT 'identity_map_entries', count(*)::text FROM identity_map
UNION ALL SELECT 'embedding_queue_pending', count(*)::text FROM pgmq.q_embedding_jobs WHERE read_ct = 0
ORDER BY metric;
```

### Target Metrics (green/warning/critical)

| Metric | Green | Warning | Critical |
|--------|-------|---------|----------|
| Empty page_content (any table) | 0 | 1-5 | >5 |
| Null sector (companies) | 0 | 1-10 | >10 |
| Embedding coverage | 100% | >95% | <95% |
| Companies duplicates found | 0 new | 1-5 new | >5 new |
| Embedding queue pending | 0-10 | 11-100 | >100 |

## 7. Enrichment Loop Template

When the Orchestrator sends `datum_enrichment` or when running on schedule:

```
1. CHECK REGRESSIONS: Query for empty page_content, null sectors
   → Fix any found (Step 5 patterns)

2. RUN MAINTENANCE: datum_daily_maintenance()
   → Log all non-OK results

3. DEDUP SWEEP: datum_company_name_deduplicator()
   → Auto-merge batch suffix duplicates (Step 2)
   → Create datum_requests for medium-confidence matches

4. SCORECARD: Run full scorecard query (Step 6)
   → Compare vs last run (store in state/datum_last_log.txt)
   → Log deltas

5. REPORT: Write structured ACK with before/after metrics
```
